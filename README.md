# Mapp AR — Flutter + Mapbox

Mapp AR is a first working version of a smart map and simulated AR navigation app built with Flutter and Mapbox.

## Features included

- Fullscreen Mapbox map for Android and iOS.
- Current location marker using live GPS.
- Smooth recenter/follow camera.
- Map styles: Streets, Satellite, Outdoors, Dark, Light.
- Zoom, compass reset, and recenter controls.
- Speed, accuracy, altitude, heading, coordinates, and poor GPS signal warning.
- Mapbox place search with recent-search cache.
- Mapbox Directions route planning and route line drawing.
- Driving, walking, and cycling route modes.
- Draggable navigation dashboard with ETA, distance remaining, speed, profile, save, start/stop.
- Simulated AR navigation overlay using Flutter widgets, compass heading, route progress, and turn guidance.
- Saved places with Home, Work, Favorite, Custom categories.
- Trip recording with route path, distance, duration, max speed, and average speed.
- Trip history map viewer.
- Offline banner and local data access for saved places, searches, and trips.
- Settings for dark mode, map style, distance/speed units, Mapbox token, and cache clearing.

## Important token rule

Do not hardcode your Mapbox token in Dart files for production.

This patch removes all `.env` / `flutter_dotenv` asset loading. That fixes Flutter Web 404 errors like:

```text
Flutter Web engine failed to fetch "assets/.env"
```

Provide the token at run/build time with `--dart-define`:

```bash
flutter run --dart-define=MAPBOX_ACCESS_TOKEN=pk.your_public_token_here
flutter build apk --dart-define=MAPBOX_ACCESS_TOKEN=pk.your_public_token_here
flutter build ios --dart-define=MAPBOX_ACCESS_TOKEN=pk.your_public_token_here
```

For local testing, you can also open the app and paste the token in **Settings → Mapbox token**. It is saved locally with SharedPreferences.

Mapbox public mobile tokens are client-side config, but still use your own restricted token for production.

## How to create the project

Because Flutter native scaffolding is generated by your installed Flutter SDK, create the shell first:

```bash
flutter create mapp_ar
cd mapp_ar
```

Then copy the files from this package into the generated project, replacing:

```text
pubspec.yaml
lib/
.gitignore
android/app/src/main/AndroidManifest.xml
```

For iOS, merge the location permission keys from:

```text
ios/Runner/Info.plist.example
```

into your generated:

```text
ios/Runner/Info.plist
```

For Android, make sure your generated `android/app/build.gradle.kts` has at least:

```kotlin
minSdk = 23
```

A helper snippet is included at:

```text
android/app/build.gradle.kts.example
```

## Install dependencies

```bash
flutter pub get
```

## Android setup

1. Ensure Android min SDK is 23 or higher in `android/app/build.gradle.kts`.
2. Ensure `android/app/src/main/AndroidManifest.xml` contains:

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
```

3. Run on a real Android device for GPS/compass accuracy:

```bash
flutter run
```

## iOS setup

1. Open `ios/Runner/Info.plist`.
2. Add these keys:

```xml
<key>NSLocationWhenInUseUsageDescription</key>
<string>Mapp AR needs your location to show your position, speed, heading, and route guidance.</string>
<key>NSLocationAlwaysAndWhenInUseUsageDescription</key>
<string>Mapp AR uses location while navigation is active.</string>
```

3. Install pods:

```bash
cd ios
pod install
cd ..
```

4. Run on a real iPhone for GPS/compass accuracy:

```bash
flutter run
```

## Patch notes for build errors

This package fixes these reported build/analyzer issues:

- Removed all `.env` / `flutter_dotenv` asset loading.
- Removed config assets from `pubspec.yaml` so Flutter Web will not fetch `assets/.env`.
- Fixed `num` to `double` inference in navigation and AR heading math.
- Replaced deprecated `cameraOptions` with `CameraViewportState` via `viewport`.
- Replaced deprecated `Color.value` with `toARGB32()`.
- Replaced deprecated `withOpacity()` with `withValues(alpha: ...)`.

## Notes

- `mapbox_maps_flutter` v2 is Android/iOS focused. This project is mobile-first.
- The AR screen is a simulated AR overlay, not camera-based ARCore/ARKit yet. It is intentionally clean and expandable.
- For real AR, the next step is adding `ar_flutter_plugin` or native ARCore/ARKit integration and placing route anchors in camera space.
- Offline support here means local app data remains usable offline. Full offline Mapbox tile packs can be added later with Mapbox offline APIs.
